# CLAUDE.md — Tank Wiki

## Назначение

Вики об автомобиле **Tank 500** (GWM/Great Wall Motors) для казахстанского сообщества владельцев.
Посвящена **только Tank 500** — отдельные страницы о других моделях Tank (300, 400, 700) не ведутся.
Основные источники — архивы Telegram-группы TANKZ Club и официальная документация; со временем добавятся Drive2, Drom, отобранные YouTube-видео и др.

Все страницы пишутся на русском (источники могут быть на других языках). LLM — единственный автор: пишет и поддерживает все страницы. Пользователь добавляет источники, задаёт вопросы, направляет процесс.

---

## Главное правило: вики-first при ответах

При **любом** вопросе про Tank 500 — это операция **Запрос (query)**, а не свободный ответ из памяти.

**Обязательный первый шаг:** прочитать `wiki/index.md`, затем релевантные страницы вики. Ответ
формировать **на основе вики**, со ссылками на страницы-источники.

**Fallback (вариант Б)** — общие знания / `web_search` — только когда в вики ответа нет или он
неполный. При этом ОБЯЗАТЕЛЬНО:
- Явно пометить в ответе, что вики не покрывает вопрос, напр.:
  `⚠️ В вики нет данных по этому вопросу — отвечаю из общих знаний (вариант Б, вики не подтверждает).`
- При частичном покрытии — разделить, что из вики, а что из fallback.
- Зафиксировать пробел в `wiki/log.md` как задачу.

Детальный алгоритм — ниже, «Операция: Запрос».

## Правила атрибуции

- Все факты относятся к **Tank 500**. Источник по умолчанию — `raw/tankz-club/tank-500/`;
  официальные документы — по явному указанию (manual/price T500).
- **Сравнительные упоминания** других моделей Tank (300/400/700) **допускаются и сохраняются**, когда
  информируют владельца T500: совместимость деталей («пружина от T300 встаёт на T500»), отличия поведения
  («в отличие от T300, LKA на T500 включается заново»), применимость прошивок и т.п. Это факты про T500.
- **Отдельные страницы, хабы и источники о других моделях не создаются.** Источник исключительно о другой
  модели в эту вики не разбирается.
- Факт о расходниках/интервалах ТО нельзя записывать без указания **варианта** (3.0T / Hi4), если он
  различается между вариантами.

### Правила размещения

| Контент | Место |
|---------|-------|
| Про сам автомобиль (ТО, проблемы, тюнинг, эксплуатация, мультимедиа, аксессуары, осмотр б/у, цены) | `wiki/{тема}/` |

**Тайбрейкер**: контент про саму машину → тематическая папка в корне (`wiki/{тема}/`);

Это каноническое правило размещения — операции ниже ссылаются на него, не повторяя.

## Типы страниц и frontmatter

### Лид-выжимка (перевёрнутая пирамида)

**Любая** длинная (≥~100 строк) или накапливающая датированную хронологию страница — **независимо от
типа** — открывается короткой лид-выжимкой текущего состояния сразу после H1: blockquote вида
`> **Коротко (YYYY-MM):** …` (дата = `последнее_обновление`/`последнее_упоминание`; если их нет —
самая свежая дата в теле, иначе без даты). Лид — самостоятельный дайджест: суть/что выбрать/чего
ожидать; не дублирует заголовки разделов, выводится из содержимого страницы. Объём пропорционален
длине (1 предложение — короткая, абзац — большая). Хронологию **не разворачивать** (нарратив читается
старое→новое), существующие секции «Выводы»/«Статус» не удалять — лид лишь поднимает ответ в первый
экран. Для типа **проблема** лид обязателен (структура: Коротко → Симптомы → … → Статус → Источники).
Короткие how-to, справочник, документы, контакты дилеров и страницы-источники в лиде не нуждаются.

### Базовый frontmatter (контентные страницы)

Применяется к типам `обслуживание`, `эксплуатация`, `проблемы`, `тюнинг`, `аксессуары`, `мультимедиа`,
`покупка`, `документы`:

```yaml
---
тип: <тип>
заголовок: ""
теги: []
---
```

Вики целиком про Tank 500, поэтому поле модели не нужно; конкретный вариант (3.0T / Hi4)
уточняется в тексте. Отличия от базы:

| тип | доп. поля сверх базы |
|-----|----------------------|
| обслуживание, эксплуатация, аксессуары, покупка, мультимедиа | — |
| проблема | `системы: []`, `статус: "открыта"` (открыта/решено/частично), `первое_упоминание: ""`, `последнее_упоминание: ""` |
| тюнинг | `поставщики: []` |
| документы | `регион: ""` (КЗ / РФ / общее) |
| справочник | без `теги` — только `тип`, `заголовок` |

- **Проблема** — кластер повторяющихся проблем по одной системе. Структура: **Коротко (лид-выжимка)**
  → Симптомы → Диагностика → Решения → [хронология] → Статус → Источники. Страница **открывается**
  короткой выжимкой текущего состояния сразу после H1 — blockquote вида
  `> **Коротко (YYYY-MM):** …` (дата = `последнее_упоминание`): текущий статус + рабочее решение +
  чего ожидать. Объём лида пропорционален длине страницы (1 предложение для коротких, абзац для
  больших). Хронологические логи `## Статус обновления (YYYY-MM)` остаются в порядке старое→новое;
  итоговый `## Статус` — внизу. Принцип «перевёрнутой пирамиды»: вывод в первом экране, история ниже.
- **Обслуживание** — регламент, марки масел, интервалы, стоимость, опыт владельцев.
- **Тюнинг** — модификации: что, сколько стоит, где, как, отзывы.
- **Аксессуар** — товарный кластер: совместимость, цены, где купить, как поставить, подводные камни.
- **Эксплуатация** — расход, режимы движения, бездорожье, зима, буксировка, прицепы, лайфхаки.
- **Покупка** — выбор комплектации, ввоз из Китая (процедура, стоимость), осмотр б/у, тренды цен КЗ.
- **Мультимедиа** — ГУ, прошивки (OTA и ручное обновление), приложения (GWM APP, дублирование экрана),
  навигация, камеры, настройки БК.
- **Документы** — растаможка, учёт, страхование (КАСКО для параллельного импорта), гарантия, техосмотр,
  утильсбор.
- **Справочник** — глоссарий сленга/аббревиатур и FAQ.

### Модель (`wiki/tank-500.md`, одна страница)

Хаб: характеристики, варианты (3.0T / Hi4 / Hi4-Z), комплектации, известные проблемы (ссылки), рекомендации.

```yaml
---
тип: модель
название: "Tank 500"
производитель: GWM
платформа: "3.0T / Hi4 / Hi4-Z"
теги: [tank-500, gwm, внедорожник]

---
```

### Дилер/Сервис (`wiki/общее/сервис-и-дилеры/*.md`, одна страница на город)

Официальные и неофициальные сервисы, контакты, репутация.

```yaml
---
тип: дилер
город: ""
теги: []
последнее_обновление: ""
---
```

Формат контактной записи:
```
### Название / @хендл
- Специализация: [что делает]
- Город/доставка: [откуда работает]
- Телефон: +7 XXX XXX XXXX (если публично указан)
- Впервые упомянут: YYYY-MM
- Последнее упоминание: YYYY-MM
- Репутация: нейтрально / позитивно / негативно
```

### Источник (`wiki/источники/*.md`)

Создаётся при разборе. Период, темы, созданные/обновлённые страницы.

```yaml
---
тип: источник
файл: ""
период: ""
источник: ""              # tankz-club-tank-500 / tankz-club-service-campaign / official / drive2 / youtube / ...
категория: ""             # telegram / pdf / video / article
сообщений_примерно: 0    # для telegram; для pdf — страниц_примерно
изображений_обработано: 0 # оценено при разборе
изображений_добавлено: 0  # прошло фильтр и добавлено в wiki/assets
страниц_создано: []
страниц_обновлено: []
дата_разбора: ""
---
```

## Организация страниц по вариантам Tank 500

Модель одна, отдельных страниц по моделям нет. Когда факт зависит от **варианта** (3.0T vs Hi4/Hi4-Z) или
исполнения (Black Edition, Black Trail):

| Если... | Решение |
|---------|---------|
| Контент существенно различается между вариантами | Отдельные страницы или H2-секции по вариантам |
| Общий симптом, разный механизм по вариантам | Одна страница с H2-секциями (`## 3.0T`, `## Hi4 / Hi4-Z`) |
| Контент одинаков для всех вариантов | Одна страница без разделения |
| Нюанс специфичен для исполнения | Inline-пометка **(Black Edition)** / **(Black Trail)** после факта |

**Inline-атрибуция**: отдельный факт о конкретном варианте/исполнении — помечать явно. Пример: «LKA
включается при каждом запуске; на **(Black Edition)** настройка запоминается». Сравнительные упоминания
других моделей оформляются так же inline («в отличие от T300…») — они остаются текстом про T500.

## Именование файлов

- Строчные буквы (кириллица и латиница), разделитель слов — дефис: `коробка-передач.md`. Без пробелов и
  спецсимволов кроме дефиса.
- Источники Telegram: `tankz-club-[топик]-YYYY-MM.md` (`tankz-club-tank-500-2024-01.md`)
- Источники official: `official-[тип]-YYYY-MM.md` (`official-manual-2026-05.md`)
- Общий паттерн источников: `[источник]-[идентификатор].md`
- Дилеры: по городу — `алматы.md`, `астана.md`

### Изображения

**Raw** (`raw/[источник]/media/`):
- Telegram: `YYYY-MM-NNN.jpg` — порядковый номер в месяце (`raw/tankz-club/tank-500/media/2025-03-014.jpg`)
- Другие источники: `[идентификатор]-NNN.jpg` (`raw/drive2/media/tank500-review-001.jpg`)

**Wiki-assets** (`wiki/assets/[тема]/`):
- По содержанию, не по источнику: `[предмет]-[уточнение].jpg`, строчные, дефис, без пробелов, макс. ~60 символов
- Примеры: `топливная-система-прорыв-трубки.jpg`, `подвеска-kman-285-70r17.jpg`
- Форматы: `.jpg`, `.png` (PNG — только для схем/текста, где важна чёткость)

## Перекрёстные ссылки

Формат — Obsidian wikilinks по **голому имени файла** (имена страниц глобально уникальны, путь не нужен):
- Модельный контент: `[[коробка-передач|Коробка передач]]`
- Общий контент: `[[алматы|Алматы]]`
- Изображения: `![[имя-файла.jpg]]` (тоже голое имя)

Правила:
1. Каждая страница связывается с хабом `tank-500` и со смежными темами (проблема↔ТО↔тюнинг↔
   эксплуатация↔мультимедиа↔покупка↔документы), когда это информирует читателя (workaround, баг прошивки,
   ввоз→растаможка и т.п.). Хаб `tank-500` держит сводные ссылки на связанные страницы.
2. Контакты хранятся только в `общее/сервис-и-дилеры/` — из других страниц давать ссылку, не дублировать.
3. Страница источника → список всех затронутых wiki-страниц.
4. Новая страница → запись в `index.md`.

## Операции

Процедурные операции (разбор, дополнение медиа, проверка) вынесены в project-skills
(`.claude/skills/`) — грузятся по требованию, вызываются через Skill tool по описанию. Подробный
алгоритм каждой — в теле соответствующего skill (включая фильтр полезности изображений для
ingest/media). Здесь — только индекс:

| Операция | Skill | Когда вызывать |
|----------|-------|----------------|
| Разбор (ingest) | `wiki-ingest` | Обработка нового raw-источника (месяц Telegram, офиц. PDF) в вики |
| Дополнение медиа | `wiki-media-supplement` | Появились изображения для уже разобранного источника (ре-экспорт с медиа) |
| Проверка (lint) | `wiki-lint` | Каждые ~5 разборов или по запросу |

**Запрос (query)** — без skill, выполняется inline (ниже): это самый частый режим.

## Операция: Запрос (query)

1. Прочитать `wiki/index.md` — найти релевантные страницы
2. Прочитать найденные страницы
3. Синтезировать ответ **только на основе вики**, со ссылками на вики-страницы
4. **Нет ответа в вики / он неполный → fallback (вариант Б):** ответить из общих знаний / `web_search`,
   **явно пометив** («⚠️ в вики нет — вариант Б, вики не подтверждает»); при частичном покрытии
   разделить, что из вики, а что из fallback
5. Пробел в знаниях → зафиксировать в `wiki/log.md` как задачу
6. Если ответ ценен как самостоятельная страница — предложить сохранить

## Ограничения

1. Raw — только чтение
2. Язык вики — русский (техн. аббревиатуры на латинице: TPMS, Hi4, ARB, GWM)
3. Telegram-хендлы можно указывать; телефоны — только если публично указаны в чате
4. Нейтральный тон о дилерах — «пользователи жаловались», не «плохой дилер»
5. Конкретность > полнота — один точный факт с датой лучше трёх обобщений
6. Все цены — с датой
7. Вики публична (см. `wiki/о-вики.md`): не добавлять новые личные телефоны/ФИО сверх минимально
   полезного; перс. данные и отзывы — под takedown-политику
