# Операция: Дополнение медиа (media supplement)

Когда изображения появились для уже разобранного источника (ре-экспорт Telegram с медиа). Текст raw при этом
НЕ перечитывается — используется только список страниц из источника.

## Опирается на CLAUDE.md (всегда в контексте)

- **Именование файлов → Изображения** — как называть файлы в `wiki/assets/{тема}/`.
- **Правила размещения** — какой теме принадлежит изображение.

Фильтр полезности изображений включён ниже (копия — это ядро операции).

## Алгоритм

1. Прочитать страницу источника `wiki/источники/[источник]-[идентификатор].md` → список затронутых страниц
2. Прочитать изображения из `raw/[источник]/media/`, применить фильтр полезности
3. Для прошедших → определить целевую wiki-страницу по содержимому
4. Прочитать целевые страницы
5. Добавить embeds в нужные места, скопировать файлы в `wiki/assets/{тема}/`
6. Обновить frontmatter источника: `изображений_обработано`, `изображений_добавлено`
7. Запись в `log.md`: `## [YYYY-MM-DD] media-supplement | [источник]-[идентификатор]`

# Изображения

## Фильтр полезности

При разборе LLM оценивает каждое изображение: включить или отбросить.

**ВКЛЮЧИТЬ:**

| Приоритет | Категория | Примеры |
|-----------|-----------|---------|
| P1 | Техническая доказательная база | Показания панели/БК (расход, ошибка), фото дефекта, part number, схемы/чертежи, фрагмент офиц. документа |
| P2 | Процедуры и измерения | Пошаговая установка, результат монтажа (зазоры/размеры), измерение щупом/манометром, до/после |
| P3 | Идентификация | Фото товара (модель/вариант), VIN-этикетка, расположение компонента |

**ИСКЛЮЧИТЬ:**

| Категория | Примеры |
|-----------|---------|
| D1 социальное | Мемы, стикеры, скриншоты переписок, селфи, фото «забрал машину», церемонии |
| D2 качество | Размытые/тёмные/нечитаемые, дубликаты, фото машины снаружи без привязки к дефекту/модификации |
| D3 коммерческое | Скриншоты маркетплейсов (цена уже в тексте), рекламные баннеры, сравнительные таблицы с сайтов |
| D4 без контекста | Фото без текстового контекста, видео-превью |

**Пограничные**: читаемый текст (номер/код ошибки), не захваченный в сообщении → ВКЛЮЧИТЬ. Информация
полностью передана текстом → ИСКЛЮЧИТЬ. 3+ похожих фото в серии → оставить одно самое информативное.

## Встраивание

```markdown
![[assets/проблемы/топливная-система-прорыв-трубки.jpg]]
*Прорыв топливной трубки, передний бак Black Edition, 2025-01 (@Ali_Ayatov)*
```

1. Embed — сразу после утверждения, которое подтверждает (не в отдельном разделе «Фото»)
2. После embed — курсивная подпись: что показано + дата + источник (хендл/документ)
3. Мягкий лимит ~5 изображений на страницу; больше — оставить самые ценные
4. Формат — Obsidian wikilink `![[assets/тема/имя.ext]]`
5. Дилерские страницы — без фото людей и зданий
