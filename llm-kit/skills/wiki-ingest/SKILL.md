# Операция: Разбор (ingest)

Перенос новых фактов из одного raw-источника в синтез вики.

## Опирается на CLAUDE.md (всегда в контексте)

Эта процедура использует разделы-схемы из `CLAUDE.md` — читать их там, не дублировать:
- **Правила размещения** (Структура директорий → таблица + тайбрейкер) — куда класть факт.
- **Типы страниц и frontmatter** — какой frontmatter у создаваемой страницы.
- **Правила атрибуции** — что относится к Tank 500, как оформлять сравнительные упоминания.
- **Именование файлов** (вкл. именование изображений) — как называть страницы и assets.
- **Организация страниц по вариантам** — когда делить 3.0T / Hi4 / Hi4-Z, inline-пометки.
- **Перекрёстные ссылки** — обязательные wikilinks (хаб, смежные темы, источник→страницы).

Именование источников, команды `filter_t500.py` и приоритет разбора — в секции
«Источники: именование, фильтрация, приоритет» ниже. Текущий статус разбора —
`wiki/источники/_статус-разбора.md`.

Фильтр полезности изображений включён ниже (копия — нужен на шагах 2а и 6).

## Принципы

Один файл за раз (файлы 40–230 КБ, до 2000+ сообщений); вики хранит синтез, не транскрипт
(повторяющаяся тема → одна запись с обновлениями); raw — ТОЛЬКО ЧТЕНИЕ, не модифицировать.

## Алгоритм

1. **Прочитать `wiki/index.md`** — карта существующих страниц до чтения raw.
2. **Прочитать raw-файл** — полностью или крупными блоками (для >150 КБ — в 2–3 прохода).
   - **2а. Оценить изображения** (если есть `raw/[источник]/media/`): после текстового разбора прочитать
     изображения этого файла; применить фильтр полезности; для прошедших — записать номер файла +
     однострочное описание. Видео пропустить.
3. **Отфильтровать шум** (НЕ в вики): приветствия, эмодзи, односложные ответы; `_[media — download disabled]_`
   без контекста; повтор без новой информации; личная переписка без итога; изображения категорий D1–D4.
4. **Извлечь факты** (в вики): вопрос + конкретный ответ (процедура/совет); личный опыт с выводом
   («поставил X — вот результат»); контакт с конкретикой (@хендл, цена, специализация); техданные (спеки,
   версии прошивок, пробеги); цена с контекстом; решение проблемы; полезное изображение (P1–P3).
5. **Сопоставить с вики-страницами** — для каждого факта:
   1. Убедиться, что факт относится к Tank 500 (или это сравнительное упоминание для владельца T500).
      Факт исключительно о другой модели — не записывать.
   2. Определить тему и при необходимости вариант (3.0T / Hi4).
   3. Разместить по правилу размещения (см. CLAUDE.md → «Структура директорий»).
   4. Читать конкретную страницу только когда факт реально на неё попадает. Нет страницы — создать.
   5. Для изображений: целевая страница по содержимому + контексту; путь `wiki/assets/{тема}/`.
6. **Обновить/создать страницы**: не дублировать (факт уже есть → обновить только при новой информации);
   противоречия — фиксировать оба варианта с источниками; цены всегда с датой («700 тыс тг (2025-01)»);
   изображения — скопировать в assets, добавить embed + подпись.
7. **Создать страницу источника** `wiki/источники/[источник]-[идентификатор].md` — указать
   `изображений_обработано` и `изображений_добавлено`.
8. **Обновить `index.md`, `log.md` и `wiki/источники/_статус-разбора.md`** (отметить разобранный
   источник: статус и число файлов).

## Особенности telegram-источника

- Медиа: доступны (`media/`) → шаг 2а; только плейсхолдер `_[media — download disabled]_` → использовать
  контекст окружающих сообщений
- "Unknown" как username → полагаться на контекст диалога (пользователь удалён из Telegram)
- Сленг → часто используемый добавлять в `wiki/общее/справочник/аббревиатуры.md`
- Уточнять вариант (3.0T / Hi4 / Hi4-Z) и исполнение (Black Edition, BT); упоминания других моделей — только
  как сравнительный контекст (см. CLAUDE.md → «Правила атрибуции»)
- Цены в тенге (тг), всегда с датой; регион: КЗ (Алматы, Астана…), иногда РФ — отмечать географию

## Правило контактов

Добавлять контакт в `общее/сервис-и-дилеры/`, если: упомянут ≥2 раз в разных месяцах, ИЛИ упомянут 1 раз с конкретикой
(цена, телефон, описание услуги).

## Правило повторных тем

Первая встреча → создать страницу. Повторная → обновить только при дельте (новая цена, уточнение,
коррекция). Чистое повторение → добавить месяц в список источников внизу страницы, контент не дублировать.

## Источники: именование, фильтрация, приоритет

Общий паттерн именования файлов источников — в CLAUDE.md → «Именование файлов». Ниже — топик-специфика,
команды фильтрации и порядок разбора. Текущий статус — `wiki/источники/_статус-разбора.md`.

- **Telegram** (`raw/tankz-club/`): топики `tank-500/` (основной), `service-campaign-tank-300-500/`
  (касается T500) и `tech-questions-t500/` (pre-filtered версия смешанного топика, только T500-треды).
  Файлы внутри: `YYYY-MM.md`. Страницы источников: `tankz-club-[топик]-YYYY-MM.md` (по месяцам) или
  `tankz-club-[топик]-YYYY.md` (по годам, если исходник объединяет несколько месяцев).
  Топики исключительно о других моделях не разбираются.
  `tech-questions/` — только чтение; для разбора использовать `tech-questions-t500/` (filtered).
  `general/` — только чтение; для разбора использовать `general-t500/` (filtered).
  `wheels-discs-tyres/` — только чтение; для разбора использовать `wheels-discs-tyres-t500/` (filtered).
  Скрипт: `tools/filter_t500.py`.
  Команды фильтрации:
  - `python3 tools/filter_t500.py raw/tankz-club/tech-questions/ --out raw/tankz-club/tech-questions-t500/`
  - `python3 tools/filter_t500.py raw/tankz-club/general/ --out raw/tankz-club/general-t500/`
  - `python3 tools/filter_t500.py raw/tankz-club/wheels-discs-tyres/ --out raw/tankz-club/wheels-discs-tyres-t500/`
  Страницы источников для general: `tankz-club-general-YYYY-MM.md`.
  Страницы источников для wheels-discs-tyres: `tankz-club-wheels-discs-tyres-YYYY.md`.
- **Официальные** (`raw/official/`): PDF от GWM/дилеров; используются документы T500 (manual, price).
  Raw: `[модель]-[тип]-YYYY-MM.pdf`. Страницы источников: `official-[тип]-YYYY-MM.md`.
- **Другие**: создать подпапку в `raw/` (`raw/youtube/`, `raw/drive2/`); страницы источников
  `[источник]-[идентификатор].md`; данные интегрировать в существующие тематические страницы (только факты
  о Tank 500).
- **Приоритет следующих**: новые официальные документы T500 → выше неформальных; затем Drive2 / YouTube /
  Drom по T500 по мере появления.

# Изображения

## Фильтр полезности

При разборе LLM оценивает каждое изображение: включить или отбросить.

**ВКЛЮЧИТЬ:**

| Приоритет | Категория | Примеры |
|-----------|-----------|---------|
| P1 | Техническая доказательная база | Показания панели/БК (расход, ошибка), фото дефекта, part number, схемы/чертежи, фрагмент офиц. документа |
| P2 | Процедуры и измерения | Пошаговая установка, результат монтажа (зазоры/размеры), измерение щупом/манометром, до/после |
| P3 | Идентификация | Фото товара (модель/вариант), VIN-этикетка, расположение компонента |

**ИСКЛЮЧИТЬ:**

| Категория | Примеры |
|-----------|---------|
| D1 социальное | Мемы, стикеры, скриншоты переписок, селфи, фото «забрал машину», церемонии |
| D2 качество | Размытые/тёмные/нечитаемые, дубликаты, фото машины снаружи без привязки к дефекту/модификации |
| D3 коммерческое | Скриншоты маркетплейсов (цена уже в тексте), рекламные баннеры, сравнительные таблицы с сайтов |
| D4 без контекста | Фото без текстового контекста, видео-превью |

**Пограничные**: читаемый текст (номер/код ошибки), не захваченный в сообщении → ВКЛЮЧИТЬ. Информация
полностью передана текстом → ИСКЛЮЧИТЬ. 3+ похожих фото в серии → оставить одно самое информативное.

## Встраивание

```markdown
![[assets/проблемы/топливная-система-прорыв-трубки.jpg]]
*Прорыв топливной трубки, передний бак Black Edition, 2025-01 (@Ali_Ayatov)*
```

1. Embed — сразу после утверждения, которое подтверждает (не в отдельном разделе «Фото»)
2. После embed — курсивная подпись: что показано + дата + источник (хендл/документ)
3. Мягкий лимит ~5 изображений на страницу; больше — оставить самые ценные
4. Формат — Obsidian wikilink `![[assets/тема/имя.ext]]`
5. Дилерские страницы — без фото людей и зданий
