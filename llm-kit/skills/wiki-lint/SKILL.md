# Операция: Проверка (lint)

Запускать каждые ~5 разборов или по запросу. Результат → запись в `log.md`. Проверять:

1. Сироты — страницы без входящих ссылок
2. Цены без даты
3. Противоречия между страницами
4. Битые wikilinks (в т.ч. на удалённые/воссозданные страницы других моделей — `tank-300`, `tank-700` и т.п.)
5. Пустые категории в `index.md`
6. Дублирование фактов
7. Контакты, упомянутые один раз без подтверждения
8. Embed без подписи — `![[assets/...]]` без курсивной строки `*...*` после него
9. Сироты в assets — файлы в `wiki/assets/` без embed-ссылки ни на одной странице
10. Необработанные изображения — файлы в `raw/*/media/` без записи в источнике
11. Контент не про Tank 500 — страница целиком о другой модели (а не сравнительное упоминание)
12. Файл в чужом расположении. Допустимые места: тематические папки про машину в корне
    (`wiki/обслуживание/`, `wiki/эксплуатация/`, `wiki/проблемы/`, `wiki/тюнинг/`, `wiki/аксессуары/`,
    `wiki/мультимедиа/`, `wiki/покупка/`), `wiki/общее/`, `wiki/источники/`, `wiki/assets/`, плюс
    файлы верхнего уровня `index.md`, `log.md`, `tank-500.md`. Воссозданная обёртка `wiki/tank-500/`,
    папка `wiki/модели/` или папка чужой модели (`wiki/tank-300/` и т.п.) — ошибка структуры.

После проверки — записать результат в `wiki/log.md`:
`## [YYYY-MM-DD] lint` + список найденных проблем (или «чисто»).

## Инструмент: grep/find (shell = zsh)

Shell здесь **zsh**, и он раскрывает незакавыченные glob'ы в аргументах. `grep -r --include=*.md`
падает с `no matches found: --include=*.md`, **обрывая команду и выдавая пустой результат** — а это
читается как ложное «0 битых ссылок» / «всё сироты». Поэтому:

- **Всегда квотировать glob**: `grep -r --include='*.md'` (или `find . -name '*.md' -print0 | xargs -0 grep`).
- **Sanity-check перед выводом**: после сборки списка ссылок/страниц проверить, что он не пустой и
  содержит заведомо известный элемент (напр. `grep -c '^tankz-club-tank-500-2024-01$' список ≠ 0`).
  Пустой список — это сбой команды, а не «чисто».
- **Отфильтровать ложные срабатывания** перед записью в log.md:
  - п.4/п.8: embed-картинки `![[...]]` — это не page-ссылки; их basename'ы (`*.jpg`/`*.png`) не должны
    попадать в проверку битых wikilinks.
  - п.1/п.4/п.8: `log.md` содержит примеры-фрагменты (`[[активаторы]]`, `![[assets/...имя.ext]]`) —
    это документация, не реальные ссылки; исключать из подсчёта.
  - п.5: `## Tank 500` и `## Общее` в index.md — хабы с H3-подразделами, а не пустые категории.
  - п.2: цена с датой в ближайшем заголовке H2/H3 над строкой — приемлемо, не нарушение.
